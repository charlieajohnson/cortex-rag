## System Overview

## Data Models

### SQLite Tables (via SQLAlchemy)

```sql
-- Tracks uploaded documents
CREATE TABLE documents (
    id          TEXT PRIMARY KEY,  -- uuid4
    filename    TEXT NOT NULL,
    content_type TEXT NOT NULL,     -- "application/pdf", "text/plain", etc.
    size_bytes  INTEGER NOT NULL,
    chunk_count INTEGER NOT NULL DEFAULT 0,
    status      TEXT NOT NULL DEFAULT 'processing',  -- processing | ready | error
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Tracks each chunk for debuggability / citation
CREATE TABLE chunks (
    id          TEXT PRIMARY KEY,  -- uuid4
    document_id TEXT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    chunk_index INTEGER NOT NULL,  -- ordering within document
    text        TEXT NOT NULL,
    token_count INTEGER NOT NULL,
    start_char  INTEGER,           -- position in original text (nullable)
    end_char    INTEGER,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_chunks_document_id ON chunks(document_id);

-- Optional: query log for analytics / debugging
CREATE TABLE queries (
    id              TEXT PRIMARY KEY,
    question        TEXT NOT NULL,
    answer          TEXT NOT NULL,
    source_chunks   TEXT NOT NULL,  -- JSON array of chunk IDs used
    model           TEXT NOT NULL,  -- e.g. "claude-sonnet-4-5-20250514"
    latency_ms      INTEGER,
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### ChromaDB Collection

```
Collection: "document_chunks"

Each record:
  id:        chunk.id (matches SQLite chunks.id)
  document:  chunk.text
  embedding: <generated by embedding model>
  metadata:
    document_id:  str
    filename:     str
    chunk_index:  int
    token_count:  int
```

> **Embedding model**: Use `voyage-3-lite` via Anthropic's recommended provider,
> or `text-embedding-3-small` from OpenAI. ChromaDB also has a built-in
> default embedder (all-MiniLM-L6-v2) which is fine for an MVP.

## API Design

### `POST /documents`
Upload a document for ingestion.

```
Request:  multipart/form-data { file: <binary> }
Response: 201
{
  "id": "abc-123",
  "filename": "design-doc.pdf",
  "status": "processing",
  "chunk_count": 0,
  "created_at": "2025-01-15T10:30:00Z"
}
```

Pipeline (runs async after response):
1. Extract text (PyPDF2 for PDF, raw read for TXT/MD)
2. Chunk text (recursive character splitter, ~500 tokens, ~50 token overlap)
3. Store chunks in SQLite
4. Embed + upsert into ChromaDB
5. Update document status to `ready`

### `GET /documents`
List all uploaded documents.

```
Response: 200
{
  "documents": [
    {
      "id": "abc-123",
      "filename": "design-doc.pdf",
      "status": "ready",
      "chunk_count": 24,
      "created_at": "2025-01-15T10:30:00Z"
    }
  ]
}
```

### `POST /query`
Ask a question against the document corpus.

```
Request:
{
  "question": "What is the system's caching strategy?",
  "document_ids": ["abc-123"],  // optional filter; null = search all
  "top_k": 5                    // optional; default 5
}

Response: 200
{
  "answer": "The system uses a write-through cache with...",
  "sources": [
    {
      "chunk_id": "chunk-456",
      "document_id": "abc-123",
      "filename": "design-doc.pdf",
      "chunk_index": 7,
      "text": "...relevant excerpt...",
      "score": 0.87
    }
  ],
  "model": "claude-sonnet-4-5-20250514",
  "latency_ms": 1230
}
```

Retrieval flow:
1. Embed the question
2. Query ChromaDB for `top_k` nearest chunks (filtered by `document_ids` if provided)
3. Build prompt: system instructions + retrieved chunks + user question
4. Call Anthropic API
5. Return answer with source attribution

### `DELETE /documents/{id}`
Remove a document and its chunks/vectors.

```
Response: 204 (no content)
```

## Prompt Template

```python
SYSTEM_PROMPT = """You are a helpful document assistant. Answer the user's
question based ONLY on the provided context. If the context doesn't contain
enough information to answer, say so вҖ” do not make things up.

When answering, cite which source(s) you drew from using [Source N] notation."""

def build_prompt(question: str, chunks: list[dict]) -> list[dict]:
    context = "\n\n".join(
        f"[Source {i+1}] (from {c['filename']}, chunk {c['chunk_index']}):\n{c['text']}"
        for i, c in enumerate(chunks)
    )
    return [
        {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {question}"}
    ]
```

### Key UI Details

- **Document status**: green dot = ready, pulsing amber dot = processing
- **Source cards**: show filename, chunk index, similarity score (0.XX), and a
  2-line text preview with overflow ellipsis
- **Answer metadata**: model name, latency in ms, source count вҖ” displayed in
  monospace below each answer
- **Animations**: `fadeUp` for new messages, `slideIn` for document list items,
  `pulse` for thinking dots and processing status
- **Upload zone**: dashed border, drag-and-drop support, hover/drag-over states
  with accent glow

### Wiring to Backend

Replace mock data with fetch calls in `api.js`:

```javascript
// api.js
const API = import.meta.env.VITE_API_URL || "http://localhost:8000";

export async function uploadDocument(file) {
  const form = new FormData();
  form.append("file", file);
  const res = await fetch(`${API}/documents`, { method: "POST", body: form });
  return res.json();
}

export async function listDocuments() {
  const res = await fetch(`${API}/documents`);
  return res.json();
}

export async function deleteDocument(id) {
  await fetch(`${API}/documents/${id}`, { method: "DELETE" });
}

export async function queryDocuments(question, documentIds = null, topK = 5) {
  const res = await fetch(`${API}/query`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ question, document_ids: documentIds, top_k: topK }),
  });
  return res.json();
}
```

## Build Order (suggested for Claude Code)

Build in this order so each step is testable before moving on:

1. **Project scaffolding** вҖ” pyproject.toml, .env.example, config.py, folder structure
2. **Database layer** вҖ” database.py, models.py, schemas.py (run alembic init or just `Base.metadata.create_all`)
3. **Vector store wrapper** вҖ” vector_store.py (init ChromaDB, add/query/delete methods)
4. **Ingestion service** вҖ” text extraction вҶ’ chunking вҶ’ embedding вҶ’ store
5. **Document endpoints** вҖ” POST /documents, GET /documents, DELETE /documents/:id
6. **Retrieval + generation** вҖ” retrieval.py, generation.py
7. **Query endpoint** вҖ” POST /query wiring retrieval вҶ’ generation
8. **Tests** вҖ” at least ingestion + retrieval + one integration test
9. **Frontend** вҖ” React app matching Cortex design (reference CortexUI.jsx mock + design spec above). Break the monolith mock into the component files listed in the file structure.
10. **README** вҖ” setup instructions, architecture diagram, demo screenshot
